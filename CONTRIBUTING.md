# Contributing to IZOS

Thank you for your interest in contributing to IZOS!

IEOS is an open-source Linux distribution built by the community, for the community. Whether you're a developer, designer, tester, writer, or simply a Linux enthusiast, your contributions are welcome.

## Ways to Contribute

### Report Bugs

If you discover a bug, please create an issue containing:

* A clear description of the problem
* Steps to reproduce it
* Expected behavior
* Actual behavior
* IZOS version
* Hardware specifications (if relevant)

### Suggest Features

Feature requests are encouraged. Before creating a new request:

* Check existing issues
* Explain the problem your feature solves
* Describe the proposed solution

### Contribute Code

1. Fork the repository.
2. Create a new branch:

```bash
git checkout -b feature/my-feature
```

3. Make your changes.
4. Test your modifications thoroughly.
5. Commit your changes:

```bash
git commit -m "Add: short description"
```

6. Push your branch:

```bash
git push origin feature/my-feature
```

7. Open a Pull Request.

## Coding Standards

* Write clean and readable code.
* Follow existing project conventions.
* Comment complex code when necessary.
* Avoid unnecessary dependencies.
* Ensure changes do not break existing functionality.

## Testing

Before submitting a Pull Request:

* Build successfully on a clean system.
* Run all available tests.
* Verify that no new warnings or errors are introduced.

## Documentation

Documentation improvements are always appreciated.

Examples include:

* Installation guides
* Troubleshooting articles
* Tutorials
* Developer documentation
* Translations

## Code of Conduct

All contributors are expected to:

* Be respectful and professional.
* Welcome constructive feedback.
* Help maintain a friendly community.
* Avoid harassment, discrimination, or toxic behavior.

## Security Issues

Please do not publicly disclose security vulnerabilities.

Instead, contact the IZOS maintainers privately by the built in github vulnerability reporter with:

* Vulnerability details
* Impact assessment
* Reproduction steps

## License

By contributing to IZOS, you agree that your contributions will be licensed under the same license as the project.
